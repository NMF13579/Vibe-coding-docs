# Глоссарий фреймворка — для пользователя

Ниже — базовые термины, которые часто встречаются в работе с AI-фреймворком.

## A

### AI-агент
- **Определение:** цифровой помощник, который выполняет шаги по правилам проекта.
- **Как используется:** анализирует задачу, предлагает план, вносит изменения после подтверждения.
- **Основной документ:** [`agent-rules.md`](./LAYER-1/agent-rules.md)

## C

### Claude Code
- **Определение:** рабочая среда для запуска AI-агента в проекте.
- **Как используется:** позволяет вести задачи и применять этапный процесс.
- **Основной документ:** [`CLAUDE.md`](./CLAUDE.md)

### Cursor
- **Определение:** IDE, где AI-агент работает прямо рядом с файлами проекта.
- **Как используется:** запуск команд через чат и быстрые действия (`Cmd/Ctrl + K`).
- **Основной документ:** [`ONBOARDING-WIZARD.md`](./ONBOARDING-WIZARD.md)

## H

### Human-in-the-Loop
- **Определение:** правило, при котором человек подтверждает критичные решения.
- **Как используется:** без подтверждения нельзя выполнять рискованные действия.
- **Основной документ:** [`agent-rules.md`](./LAYER-1/agent-rules.md)

## I

### IDE
- **Определение:** программа для работы с проектом и документами.
- **Как используется:** в IDE открывают файлы, запускают агента и проверяют результат.
- **Основной документ:** [`QUICK-START.md`](./QUICK-START.md)

## L

### LLM
- **Определение:** большая языковая модель, на которой работает AI-агент.
- **Как используется:** обрабатывает запросы и формирует ответы/изменения.
- **Основной документ:** [`ADVANCED-SETUP.md`](./ADVANCED-SETUP.md)

## M

### MCP
- **Определение:** способ подключать внешние сервисы и инструменты к агенту.
- **Как используется:** для интеграций (браузер, базы, платформы).
- **Основной документ:** [`ADVANCED-SETUP.md`](./ADVANCED-SETUP.md)

## P

### Prompt
- **Определение:** текстовая инструкция, которую вы даёте агенту.
- **Как используется:** задаёт цель, границы и ожидаемый результат.
- **Основной документ:** [`ONBOARDING-WIZARD.md`](./ONBOARDING-WIZARD.md)

### Prompt Injection
- **Определение:** попытка подменить правила агента через вредный текст.
- **Как используется:** считается риском, который агент обязан отклонять.
- **Основной документ:** [`security.md`](./LAYER-1/security.md) — раздел «Защита от prompt injection»

## R

### Risk-Tag
- **Определение:** метка уровня риска задачи (`HIGH`, `MEDIUM`, `LOW`).
- **Как используется:** определяет глубину проверки перед завершением.
- **Основной документ:** [`task-protocol.md`](./LAYER-1/task-protocol.md)

### Rollback
- **Определение:** безопасный возврат к последнему рабочему состоянию.
- **Как используется:** применяется при сбоях, рисках и выходе за scope.
- **Основной документ:** [`error-handling.md`](./LAYER-1/error-handling.md) — раздел «Процедура отката»

## S

### Scope Guard
- **Определение:** механизм контроля границ текущей задачи.
- **Как используется:** останавливает выполнение при попытке «сделать заодно».
- **Основной документ:** [`scope-guard.md`](./LAYER-1/scope-guard.md)

### Self-Verification
- **Определение:** проверка качества и рисков до выполнения изменений.
- **Как используется:** помогает поймать проблему до ошибки в работе.
- **Основной документ:** [`self-verification.md`](./LAYER-1/self-verification.md)

## Ф

### 152-ФЗ
- **Определение:** закон о персональных данных в РФ.
- **Как используется:** задаёт требования к согласию, хранению и обработке ПДн.
- **Основной документ:** [`LEGAL-152FZ.md`](./LAYER-1/LEGAL-152FZ.md)

## П

### ПДн (персональные данные)
- **Определение:** данные, по которым можно определить человека.
- **Как используется:** требуют отдельной правовой и технической защиты.
- **Основной документ:** [`LEGAL-152FZ.md`](./LAYER-1/LEGAL-152FZ.md)
