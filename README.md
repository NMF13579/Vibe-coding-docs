# Vibe Coding Docs — как этим пользоваться

Этот репозиторий не для программистов, а для совместной работы владельца проекта и ИИ-агента.  
Главная идея: владелец объясняет простыми словами, агент читает документы и берёт на себя технику.

---

## Если ты владелец проекта (не разработчик)

1. Открой файл **`docs/OWNER-CHEATSHEET.md`** — это текстовая шпаргалка.
2. Открой **`docs/cheatsheet.html`** в браузере — это интерактивная версия со строкой поиска и кнопками «копировать».

Когда не знаешь, что писать агенту — сначала смотри туда.

### Самые полезные команды

- `Начнём` — запустить новый проект.
- `Восстанови контекст` — начать новую сессию по уже существующему проекту.
- `Сохрани контекст` — закончить сессию так, чтобы ничего не потерялось.
- `Всё верно` / `Ок` — подтвердить план, документ или следующий шаг.
- `Всё сломалось` — когда приложение перестало работать и нужно откатить изменения.
- `Проверь готовность к релизу` — финальная проверка перед показом пользователям.

---

## Как устроен процесс

- **START.md** — модель запуска: три уровня (быстрый прототип, MVP, production-MVP).  
- **docs/** — инструкции для агента (интервью, выбор стека, безопасность, тесты, релиз).  
- **memory-bank/** — «память» проекта: статус, интеграции, отложенные решения.  
- **tasks/** — задачи, которые агент делает по очереди.

Тебе как владельцу достаточно:

- В начале новой сессии писать: `Восстанови контекст`.
- В конце сессии писать: `Сохрани контекст`.
- Подтверждать важные шаги фразой `Всё верно`.

Всё остальное — работа агента: читать документы, предлагать решения простым языком и аккуратно вносить изменения в код.

# Vibe Coding Starter

> Пакет документации для управляемого вайб-кодинга.
> Агент ведёт проект от идеи до деплоя — как продакт-менеджер и технический советник.

---

## Быстрый старт

1. Скопируй этот репозиторий как шаблон для нового проекта.
2. Вставь содержимое `SYSTEM_PROMPT.md` в system prompt своего инструмента (Cursor, Claude Code, Lovable, Bolt).
3. Напиши агенту: **«Начнём новый проект»** — он сам проведёт тебя через всё остальное.

## Принципы пакета

- Агент не пишет код без подтверждённого плана — всегда.
- Документы заливаются заполненными, не пустыми шаблонами.
- Один вопрос за раз — агент не засыпает списком.
- ROADMAP защищает от расползания проекта в разные стороны.
- DECISIONS.md останавливает «переоткрытие» уже принятых решений.

## AI integrations

Этот репозиторий поддерживает несколько AI-сред для vibe coding, но основной источник знаний — сами документы проекта, а не длинный системный промпт. `CLAUDE.md` должен оставаться коротким и ссылаться на `docs/*`, потому что best practices Claude Code рекомендуют держать его concise и выносить детали в отдельные markdown-файлы.[1][2][3]

### Где лежат инструкции

- **Claude Code** → `CLAUDE.md`, `.claude/agents/*`.
- **Cursor** → `.cursor/rules/*`.
- **GitHub Copilot** → `.github/copilot-instructions.md`, `.github/instructions/*`.
- **Gemini и похожие системы** → `GEMINI.md`.

### Источник истины

- Основные правила и процессы лежат в `docs/*`.
- Текущее состояние проекта и накопленная память лежат в `memory-bank/*`.
- Tool-specific файлы только направляют AI, что читать и как работать в этом репозитории.[4][5][6]

### Что читать первым

- Для человека: `README.md` → `START.md` → `docs/OWNER-GUIDE.md`.
- Для AI: `README.md` → `START.md` → `CLAUDE.md` или соответствующий adapter-файл.
- Для продолжения работы после паузы: `HANDOFF.md` → `memory-bank/project-status.md` → `memory-bank/lessons-learned.md`.

### Принцип работы

- Документы в `docs/*` описывают процесс, архитектуру, аудит, ревью и стиль коммуникации.
- Документы в `memory-bank/*` сохраняют состояние между сессиями.
- Адаптеры для Claude, Cursor, Copilot и Gemini помогают разным AI-системам использовать один и тот же пакет без дублирования логики.[7][5][8]
