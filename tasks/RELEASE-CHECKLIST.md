# RELEASE-CHECKLIST — перед показом пользователям

> Этот чеклист проходится ПЕРЕД первым деплоем или перед показом продукта
> реальным пользователям.
> Агент не объявляет «готово к запуску» без прохождения этого списка.

---

## Блок 1 — Работоспособность
- [ ] Приложение открывается без ошибок на компьютере
- [ ] Приложение открывается без ошибок на телефоне
- [ ] Все основные сценарии из `LAYER-2/ux/USER-FLOWS.md` проверены
- [ ] Регистрация и вход работают
- [ ] Данные сохраняются и отображаются после перезагрузки

## Блок 2 — Безопасность
- [ ] Нет секретов в публичном коде (API ключи, пароли)
- [ ] Пользователь видит только свои данные
- [ ] Форма регистрации требует надёжный пароль
- [ ] HTTPS включён (не HTTP)
- [ ] `.env` файл не попал в репозиторий

## Блок 3 — Данные и база
- [ ] Бэкап базы данных настроен
- [ ] Тестовые данные удалены из продакшена
- [ ] Нет таблиц без Row Level Security (если Supabase)

## Блок 4 — Пользовательский опыт
- [ ] Сообщения об ошибках написаны понятно (не «Error 500»)
- [ ] Есть сообщение об успешном действии (сохранено, отправлено и т.д.)
- [ ] Долгие операции показывают индикатор загрузки
- [ ] Пустые состояния объяснены (что делать если нет данных)

## Блок 5 — Документация
- [ ] `CHANGELOG.md` описывает что входит в этот релиз
- [ ] `LAYER-3/project-status.md` обновлён
- [ ] `LAYER-2/specs/roadmap.md` — текущий этап отмечен выполненным

## Блок 6 — Мониторинг (если настроен)
- [ ] Логи ошибок доступны для просмотра
- [ ] Есть способ узнать что что-то сломалось (алерт / email)

---

> - Если что-то сломалось — пиши мне, используем ROLLBACK-PROTOCOL.md
> - Собирай первые отзывы пользователей для следующей итерации»
