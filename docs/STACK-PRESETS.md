# STACK-PRESETS — готовые наборы инструментов

> Агент читает этот файл когда нужно предложить стек.
> Цель: не изобретать велосипед, использовать проверенные комбинации.
> Каждый пресет — это набор инструментов которые хорошо работают вместе.

---

## Как агент выбирает пресет

После интервью агент определяет:
1. **Тип продукта** (веб / мобильное / бот / внутренний инструмент)
2. **Сложность** (лендинг / приложение с данными / платформа)
3. **Чувствительность данных** (обычные / медицинские / финансовые)
4. **Уже используемые инструменты** (что пользователь знает)

Потом предлагает **один пресет** с объяснением — не список из 10 вариантов.

---

## Пресет 1 — Веб-приложение, старт

**Когда:** веб-сервис с регистрацией, хранением данных, личным кабинетом
Интерфейс: Lovable
База данных: Supabase
Авторизация: Supabase Auth (встроено)
Деплой: Vercel (через Lovable — автоматически)
Email: Resend

text

**Агент объясняет так:**
> Это самая популярная комбинация для вайб-проектов.
> Lovable рисует интерфейс по твоим словам, Supabase хранит данные,
> Vercel выкладывает сайт. Всё это работает вместе «из коробки».
> Бесплатно до реального роста.

---

## Пресет 2 — Мобильное приложение

**Когда:** нужно приложение в App Store / Google Play
Интерфейс: FlutterFlow
База данных: Supabase
Авторизация: Supabase Auth
Push-уведомления: Firebase Cloud Messaging (FCM)
Деплой: FlutterFlow → напрямую в магазины

text

**Агент объясняет так:**
> FlutterFlow — единственный нормальный способ сделать мобильное приложение
> без программирования. Один проект — сразу iOS и Android.
> Supabase как и в веб-версии — данные, вход, хранение файлов.

---

## Пресет 3 — Telegram-бот

**Когда:** автоматизация через Telegram, уведомления, простые сервисы
Бот-логика: n8n (self-hosted) или Make.com
База данных: Supabase или Airtable
Хостинг бота: Timeweb Cloud или Railway
ИИ (если нужен): OpenAI API или YandexGPT

text

**Агент объясняет так:**
> Telegram-бот — это программа которая отвечает на сообщения.
> n8n или Make.com позволяют настроить логику без кода, визуально.
> Supabase хранит данные (история, пользователи, настройки).

---

## Пресет 4 — Лендинг / сайт-визитка

**Когда:** нужен сайт без сложной логики — рассказать о продукте, собрать заявки
Конструктор: Tilda или Lovable
Форма заявок: встроенная в Tilda / Typeform
Email: Resend или встроенный в Tilda
Деплой: встроен в Tilda / Vercel для Lovable
Аналитика: Яндекс.Метрика

text

**Агент объясняет так:**
> Если нужен просто красивый сайт с формой — Tilda самый быстрый вариант.
> Если хочешь потом добавить личный кабинет или сложную логику — лучше Lovable,
> он легко расширяется.

---

## Пресет 5 — Внутренний инструмент / дашборд

**Когда:** таблица данных, отчёты, управление контентом — для внутреннего использования
Интерфейс: Retool или Lovable
База данных: Supabase
Авторизация: Supabase Auth (только нужные люди)
Деплой: Vercel

text

**Агент объясняет так:**
> Retool создан специально для внутренних инструментов — таблицы, фильтры,
> кнопки действий. Быстрее чем делать с нуля.
> Если нужен более красивый интерфейс — Lovable.

---

## Пресет 6 — Медицинское приложение (данные в РФ)

**Когда:** данные о пациентах, диагнозах, медицинской истории
Интерфейс: Lovable или Cursor
База данных: Supabase (self-hosted на российском сервере) или Timeweb Cloud PG
Авторизация: Supabase Auth
Хостинг: Timeweb или Яндекс.Облако (данные в России — требование 152-ФЗ)
Email: Unisender (российский сервис)
Платежи: ЮКасса

text

**Агент объясняет так:**
> Медицинские данные — особая категория по российскому закону (152-ФЗ).
> Их нельзя хранить на серверах за рубежом.
> Поэтому используем российские сервисы для хранения.
> Это важно — иначе могут быть проблемы с регуляторами.

---

## Пресет 7 — RAG / ИИ-ассистент

**Когда:** чат-бот на своих документах, умный поиск, ИИ-помощник
Интерфейс: Lovable или Bolt
База данных: Supabase (с расширением pgvector для поиска)
Логика ИИ: n8n или Flowise
ИИ-модель: OpenAI API / Anthropic / YandexGPT
Хранение файлов: Supabase Storage
Деплой: Vercel + Railway (для n8n)

text

**Агент объясняет так:**
> Если нужен ИИ который отвечает на вопросы по твоим документам —
> это называется RAG. Supabase умеет хранить не просто данные,
> а «смысл» текстов — и ИИ ищет по смыслу, а не по словам.
> n8n или Flowise — это визуальный конструктор для логики ИИ.

## Когда пресет не подходит

Если ни один пресет не совпадает — агент говорит:
Твой проект чуть необычный — стандартный набор не подойдёт.
Давай разберём подробнее: [уточняющий вопрос].

text

И собирает стек вручную, объясняя каждый выбор через DECISION-GUIDE.md.
