# OWNER GUIDE — шпаргалка для владельца

> Для тебя, не для агента.
> Помогает отвечать на вопросы агента правильно с первого раза.

---

## «Что за проект?»

**Формула:**
> «Это [сайт/бот/приложение] для [кто].
> Главное что делает — [одна функция].
> Работает через [браузер/телефон/Telegram].»

---

## «Какие технологии?»

Просто скажи сервисы:

| Используешь | Говори |
|-------------|--------|
| Supabase | «База — Supabase» |
| Vercel | «Хостинг — Vercel» |
| Lovable | «Интерфейс — Lovable» |
| Telegram | «Нужен Telegram-бот» |

---

## «Какие данные хранятся?»

Перечисли да/нет:
- ФИО пациентов
- Диагнозы, назначения, анализы
- Телефоны, email
- Финансовые данные

---

## «Кто имеет доступ?»

Опиши роли словами:
> «Пациент видит только свои данные.
> Врач видит своих пациентов.
> Администратор видит всё.»

---

## «Как понять что фича работает?»

Опиши шаги в браузере:
> «1. Захожу как врач.
> 2. Открываю Пациентов.
> 3. Вижу только своих.»

---

## Правила общения с агентом

**Сначала план:** «Покажи план перед тем как писать код.»

**Непонятно:** «Объясни проще, я не программист.»

**Сделал не то:** «Ожидал X, вижу Y. Вот шаги: [...]»

**После задачи:** «Что изменил? Какие файлы? Как проверить?»

**Стоп-слово:** «Стоп. Вернись к плану.»

---

## Стартовый промпт для нового проекта

```
Начинаем новый проект. Прочитай START.md и начни процесс.
Один вопрос за раз. После каждого ответа подтверждай что понял.
После всех вопросов — покажи документы на моё одобрение.
Я не программист — объясняй простым языком.
```
