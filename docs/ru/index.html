Для кого это

Выберите роль — откроется пошаговая инструкция, как работать с AI-агентами.

Как это устроено

Четыре этапа держат описание продукта в одном месте — агенты не начинают с нуля.

Установите плагин

Одна команда (или короткая настройка) добавляет интервью, спеки и чек-листы в проект.

Опишите проект

Этап интервью фиксирует цели, пользователей и ограничения в одном «живом» документе.

Агент строит с контекстом

Разработка опирается на тот же файл — решения остаются согласованными.

Деплой с сохранённой памятью

Handoff и статус помогают следующей сессии продолжить без повторных объяснений.

Установка

Выберите инструмент и скопируйте команду. Остальное — в README репозитория.

/plugin install vibe-coding-docs@https://github.com/NMF13579/Vibe-coding-docs.git

Запустите в Claude Code. Плагин подтянет актуальные файлы фреймворка в проект.

В Cursor: Настройки → Rules — добавьте правило со ссылкой на файл CLAUDE.md в репозитории (или укажите путь). Это точка входа в тот же сценарий.

Файл на GitHub: CLAUDE.md (raw) 

https://raw.githubusercontent.com/NMF13579/Vibe-coding-docs/main/CLAUDE.md
{ "plugin": ["vibe-coding-docs@git+https://github.com/NMF13579/Vibe-coding-docs.git"] }

Вставьте фрагмент в конфигурацию OpenCode в список плагинов.

Медицинский специалист

Делайте понятные клинические помощники, задавая продуктовые факты — без программирования.

  1. Пройдите интервью. Опишите сценарий, кто пользователь и чего нельзя допустить.
  2. Один источник правды. Пусть правки идут в проектный документ — каждая сессия видит ту же историю.
  3. Проверяйте результат, а не код. Просите агента объяснить экраны и сценарии простым языком до согласования.
  4. Комплаенс с начала. Когда команда готова — фиксируйте согласие, данные и юрисдикцию в защищённых секциях.

Дизайнер

От макетов к прототипу с сохранением UX-намерения между сессиями агента.

  1. Прикрепите референсы. Экспортируйте ключевые кадры и дайте ссылки в заметках discovery.
  2. Описывайте состояния. Пустые, загрузка, ошибка, успех — так промпты остаются конкретными.
  3. Короткие циклы. Запросите экран, ответьте скрином или списком правок, обновите спеку.
  4. Ворота фреймворка. «План → ревью → подтверждение» снижает случайные переделки посреди сборки.

Менеджер

Координируйте подрядчиков или внутреннюю команду без потери решений между сессиями AI.

  1. Сначала результат. Зафиксируйте метрики успеха и сроки в интервью — агенты ориентируются на бизнес-цель.
  2. Назначьте владельцев. Кто утверждает тексты, бренд и релизы — в проектном файле.
  3. Контролируйте объём. Новые идеи заносите в «отложено», а не в тихий scope creep.
  4. Читайте handoff. Перед сессией откройте статус — увидите, что изменилось в прошлый раз.

Разработчик

Дайте вайб-кодингу структуру: контракты, слои и ревью, которые масштабируются.

  1. Установка на репозиторий. Держите `CLAUDE.md`, слои и BOOT этапов по путям шаблона.
  2. Уважайте lock. Обучите команду, какие секции `PROJECT.md` трогать только с явного согласия.
  3. Автопроверки. Направьте CI на те же smoke-тесты, что в dev-доках этапа.
  4. ADR. При смене стека дописывайте решения — иначе агенты откатят выбор.
