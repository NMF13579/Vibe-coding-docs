# VALIDATION — Объективная проверка результата

> **Порядок:** сначала `docs/TASK-REVIEW-PROTOCOL.md` (качество кода и документация), затем этот файл (объективная проверка результата).
> Цель: получить объективный ответ «работает / не работает» без субъективных оценок.

---

## Когда запускается

- Команда владельца: «проверь»
- После прохождения TASK-REVIEW-PROTOCOL перед закрытием задачи
- Когда агент сомневается что решение правильное

---

## Чеклист валидации

### Уровень 1 — Базовая работоспособность

- [ ] Основной сценарий работает от начала до конца?
- [ ] Проект запускается без ошибок в консоли?
- [ ] Данные сохраняются и читаются корректно?

### Уровень 2 — Соответствие задаче

- [ ] Результат соответствует описанию цели в TASK?
- [ ] Все критерии готовности из TASK отмечены?
- [ ] Поведение совпадает с ожидаемым в шагах задачи?

### Уровень 3 — Граничные случаи

- [ ] Что происходит если поле пустое?
- [ ] Что происходит если пользователь не авторизован?
- [ ] Что происходит при повторном выполнении действия?

> Агент проверяет только те пункты Уровня 3 которые релевантны задаче.
> Не нужно проверять авторизацию если задача про вёрстку кнопки.

---

## Мини-тесты (заполняет агент под задачу)

> Агент формулирует 2-3 конкретных теста для текущей задачи.
> Шаблон:

```
Тест 1: [действие] → ожидаем [результат] → получили [результат] → ✅/❌
Тест 2: [действие] → ожидаем [результат] → получили [результат] → ✅/❌
Тест 3: [действие] → ожидаем [результат] → получили [результат] → ✅/❌
```

---

## Результат валидации

Агент пишет одно из трёх:

**✅ PASS** — все пункты пройдены, задачу можно закрывать.

**⚠️ PASS WITH NOTES** — основное работает, есть мелкие замечания:
> [список замечаний] — зафиксированы в заметках задачи, не блокируют закрытие.

**❌ FAIL** — валидация не пройдена:
> Провальный тест: [описание]
> Действие: MODE = PLANNING. Пересобрать подход к задаче целиком.
> (RETRY RULE — только для повторной попытки закрыть один пункт чеклиста, не для полного FAIL)

---

## Правило объективности

Агент не пишет «скорее всего работает» или «должно работать».
Только: работает ✅ или не работает ❌.
Если проверить невозможно — агент пишет: «Не могу проверить без [что нужно]»
и запрашивает у владельца.
