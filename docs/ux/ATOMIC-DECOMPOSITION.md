# ATOMIC-DECOMPOSITION — порядок заполнения UX-документов

> Этот файл ведёт агент. Владелец проекта его не заполняет напрямую.
> Цель: связать процессы → экраны → блоки интерфейса → спецификацию.

## Порядок заполнения (Контур 2)

1. `ux/UX-FLOWS-DESKTOP.md` и `ux/UX-FLOWS-MOBILE.md` — сценарии из процессов
2. `ux/SCREEN-MAP.md` — карта экранов и переходов
3. `ux/PAGES.md` — каждый экран целиком
4. `ux/TEMPLATES.md` — повторяемые раскладки
5. `ux/ORGANISMS.md` → `ux/MOLECULES.md` → `ux/ATOMS.md` — декомпозиция блоков
6. `ux/WIREFRAMES.md` — схемы (текстом или ссылками)
7. `ux/UI-CHECKLIST.md` — проверка UX против процессов
8. `ux/UX-GAP-REPORT.md` — пробелы между процессом и экранами
