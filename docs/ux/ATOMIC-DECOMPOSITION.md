# ATOMIC-DECOMPOSITION — разбивка экрана на компоненты

> Этот файл — инструкция агенту по порядку заполнения UX-слоя.
> Заполняется после SCREEN-MAP.md и PAGES.md, до написания кода.

## Порядок заполнения UX-документов

1. `SCREEN-MAP.md` — список экранов и переходов между ними
2. `PAGES.md` — таблица экранов с ID, шаблоном, организмами
3. `WIREFRAMES.md` — текстовая схема каждого экрана сверху вниз
4. `ORGANISMS.md` — крупные блоки интерфейса (шапка, форма, список)
5. `MOLECULES.md` — составные элементы (строка поиска с кнопкой, карточка)
6. `ATOMS.md` — базовые элементы (кнопка, поле, иконка, метка)

## Шаблон записи компонента

| ID | Название | Состав | Состояния | Экраны |
|----|----------|--------|-----------|--------|
| ORG-001 | [название] | [из чего состоит] | default, loading, error, empty | PG-XXX |

## Правила

- Начинай с ORGANISMS — они видны сразу
- MOLECULES выводи из состава организмов
- ATOMS выводи из состава молекул
- Один элемент — одна запись, даже если встречается на нескольких экранах
- ID: ORG-XXX / MOL-XXX / ATM-XXX
