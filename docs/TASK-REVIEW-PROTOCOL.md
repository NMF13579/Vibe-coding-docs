# TASK-REVIEW-PROTOCOL — самопроверка перед «задача выполнена»

> Агент проходит этот чеклист ПЕРЕД тем как объявить задачу закрытой.
> Пропуск пунктов запрещён.

---

## Чеклист агента (обязательный)

### Блок 1 — Код
- [ ] Написанный код решает именно то, что описано в задаче
- [ ] Нет закомментированного старого кода
- [ ] Нет `console.log` с чувствительными данными
- [ ] Нет захардкоженных секретов (пароли, API ключи, токены)
- [ ] Нет TODO которые блокируют работу функции

### Блок 2 — Функциональность
- [ ] Основной сценарий работает (happy path)
- [ ] Граничные случаи обработаны (пустой ввод, длинный текст, неверный формат)
- [ ] Ошибки показываются пользователю понятно — не технический текст

### Блок 3 — Безопасность
- [ ] Новые данные пользователя защищены (RLS / auth проверка)
- [ ] Новые API эндпоинты требуют авторизации (если нужно)
- [ ] Пользователь не может увидеть чужие данные

### Блок 4 — Документация
- [ ] `tasks/TASK-XXX.md` → статус обновлён на «Выполнено»
- [ ] `HANDOFF.md` → обновлён (что сделано, следующий шаг)
- [ ] `memory-bank/project-status.md` → обновлён
- [ ] `CHANGELOG.md` → добавлена запись об изменении

### Блок 5 — Инструкция для проверки

Агент пишет владельцу конкретные шаги:
✅ Задача выполнена. Проверь так:

[Открыть / перейти куда]

[Сделать конкретное действие]

Ожидаемый результат: [что должно произойти]

Если что-то не так → используй шаблон из TESTING-GUIDE.md

text

---

## Когда задача считается закрытой

Только после того как владелец написал одно из:
- «Работает»
- «Ок» / «Окей»
- «Подтверждаю»
- «Всё верно»
- «Да» / «Приступай» / «Go» / «Выполняй»
- «✓»

**Молчание не считается подтверждением.**
**Канонический список** — этот файл. Все остальные файлы ссылаются сюда.

---

## Что делать если чеклист не проходит

Если агент обнаружил проблему при самопроверке:
1. Исправить до объявления задачи выполненной
2. Если исправление меняет scope — сообщить владельцу
3. Если исправление займёт больше 30 минут — создать новую задачу
