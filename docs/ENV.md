# ENV.md

Универсальное описание переменных окружения для проекта, пригодное для Vercel, Timeweb Cloud, Railway, Render, Docker и VPS.

## Цель документа

Этот файл объясняет:
- какие переменные нужны проекту;
- какие из них обязательные;
- где их задавать;
- какие значения нельзя хранить в git.

## Главные правила

- Никогда не хранить реальные секреты в репозитории.
- В репозитории хранить только `.env.example` без настоящих значений.
- Реальные значения задавать в панели платформы, секрет-хранилище или на сервере.
- Любое изменение env-переменных фиксировать в `HANDOFF.md` и при необходимости в `CHANGELOG.md`.

## Минимальный шаблон категорий переменных

### 1. Системные

Примеры:
- `NODE_ENV`
- `PORT`
- `APP_ENV`

Назначение:
- указывают, в какой среде работает приложение.

### 2. База данных

Примеры:
- `DATABASE_URL`
- `POSTGRES_URL`
- `REDIS_URL`

Назначение:
- подключение к хранилищам данных.

### 3. Аутентификация

Примеры:
- `NEXTAUTH_SECRET`
- `JWT_SECRET`
- `SESSION_SECRET`

Назначение:
- подпись токенов, сессий, логина.

### 4. Внешние API

Примеры:
- `OPENAI_API_KEY`
- `ANTHROPIC_API_KEY`
- `SMTP_PASSWORD`
- `S3_ACCESS_KEY`

Назначение:
- интеграции со сторонними сервисами.

### 5. Публичные переменные

Примеры:
- `NEXT_PUBLIC_API_URL`
- `PUBLIC_APP_NAME`

Правило:
- такие переменные могут попасть во frontend;
- нельзя хранить там секреты.

## Таблица переменных

| Имя | Обязательна | Где используется | Для каких сред | Секрет | Комментарий |
|---|---|---|---|---|---|
| `APP_ENV` | Да | App runtime | local, staging, production | Нет | Значение среды |
| `DATABASE_URL` | Обычно да | Backend / ORM | staging, production | Да | Подключение к БД |
| `AUTH_SECRET` | Обычно да | Auth | staging, production | Да | Секрет сессий |
| `PUBLIC_BASE_URL` | Часто да | Frontend / callbacks | staging, production | Нет | Базовый URL |

## Где задавать переменные

### Local

- файл `.env.local` или аналог;
- не коммитить;
- использовать только для разработки.

### Staging / Production

Можно задавать:
- в Vercel Project Settings;
- в Timeweb Cloud App settings;
- в Railway Variables;
- в Render Environment;
- в Docker Compose / secrets / CI variables;
- в секрет-хранилище.

## Обязательные артефакты рядом

- `.env.example`
- `docs/DEPLOY.md`
- `docs/MONITORING.md`
- `docs/ROLLBACK-PROTOCOL.md`

## Что должен делать AI-агент

Перед деплоем агент обязан:

1. Проверить, что все обязательные переменные описаны.
2. Не печатать реальные секреты в чат.
3. Не записывать секреты в git.
4. Уточнить у владельца, если обязательной переменной не хватает.
5. Не продолжать production-деплой, если обязательная переменная отсутствует.
