# ENV-SETUP — Переменные окружения для не-программиста

> Это один из самых важных базовых навыков при работе с проектом.
> Читайте спокойно — здесь нет ничего сложного.

---

## Что такое переменные окружения и зачем они нужны

Представьте: у вас есть ключ от квартиры.
Вы же не пишете его на двери — вы храните отдельно.

Токены, пароли и API-ключи — это ключи от ваших сервисов.
**Переменная окружения** — это способ хранить такой ключ отдельно от кода,
чтобы случайно не показать его всему миру.

**Почему нельзя просто вписать токен в код:**
- Код лежит в GitHub — значит токен увидят все
- Боты автоматически сканируют GitHub в поисках токенов
- Timeweb, GitHub и другие сервисы при обнаружении токена
  **автоматически его аннулируют** — и придётся создавать заново

**Как работает переменная:**
```
mcp.json видит:    "GITHUB_PERSONAL_ACCESS_TOKEN": "ВАШ_GITHUB_TOKEN"
.env хранит:       GITHUB_PERSONAL_ACCESS_TOKEN=ghp_abc123реальныйтокен
В коде и GitHub:   только название, без значения
```

---

## Способ 1 — Прямо в mcp.json (самый простой, для начала)

Если вы работаете один и репозиторий приватный — допустимо просто вписать
токен прямо в `mcp.json` и добавить файл в `.gitignore`.

**Шаг 1.** Откройте файл `.cursor/mcp.json` в Cursor

**Шаг 2.** Замените текст `ВАШ_GITHUB_TOKEN` на реальный токен:
```json
"GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_abcdef123456ваштокен"
```

**Шаг 3.** Убедитесь что файл в `.gitignore` (см. ниже)

> ✅ Этого достаточно для быстрого старта
> ⚠️ Если репозиторий публичный — используйте Способ 2

## Способ 2 — Файл .env (правильный способ)

### Что такое .env файл

`.env` — это обычный текстовый файл с особым именем.
В нём хранятся все ключи и пароли проекта.
Файл лежит в папке проекта, но **никогда не уходит в GitHub**.

### Как создать .env файл

**Вариант А — через Cursor (рекомендуется):**

1. Откройте папку проекта в Cursor
2. В левой панели (список файлов) нажмите правой кнопкой на пустое место
3. Выберите **New File**
4. Назовите файл строго: `.env` (точка в начале — это обязательно)
5. Нажмите Enter — файл создан

**Вариант Б — через терминал:**

1. Откройте терминал в Cursor: меню **Terminal → New Terminal**
2. Напечатайте и нажмите Enter:
   ```
   touch .env
   ```
3. Файл появится в списке файлов слева

> ⚠️ На Mac файлы с точкой в начале скрыты по умолчанию.
> В Cursor они всегда видны — не беспокойтесь.

### Что написать внутри .env

Откройте созданный файл `.env` и напишите в таком формате:
```
GITHUB_PERSONAL_ACCESS_TOKEN=ghp_abcdef123456ваштокен
TIMEWEB_API_TOKEN=вашTokenTimeweb
```

**Правила формата:**
- Каждая переменная на отдельной строке
- Знак `=` без пробелов вокруг него
- Никаких кавычек вокруг значения
- Строки с `#` — это комментарии, они игнорируются:
  ```
  # Это комментарий — не читается программой
  GITHUB_PERSONAL_ACCESS_TOKEN=ghp_abc123
  ```

### Как это связать с mcp.json

В `mcp.json` вместо реального токена оставьте только название переменной.
Cursor сам прочитает `.env` и подставит значение:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_PERSONAL_ACCESS_TOKEN}"
      }
    }
  }
}
```

`${НАЗВАНИЕ}` — это и есть «подставь значение из переменной с этим именем».

## Шаг обязательный — защитить .env от попадания в GitHub

**Что такое .gitignore:**
`.gitignore` — файл-список того, что **не** нужно отправлять в GitHub.
Всё что написано в нём — остаётся только у вас на компьютере.

**Как добавить .env в .gitignore:**

1. Найдите файл `.gitignore` в корне проекта
   (если его нет — создайте так же как `.env`, назовите `.gitignore`)
2. Откройте его
3. Добавьте строки:
   ```
   .env
   .env.local
   .env.production
   .cursor/mcp.json
   ```
4. Сохраните файл (`Cmd + S` или `Ctrl + S`)

> После этого Git будет полностью игнорировать `.env`.
> Даже если вы нажмёте «закоммитить всё» — `.env` не уйдёт в GitHub.

## Создать .env.example для команды

Если вы работаете с командой или хотите сохранить
**шаблон без реальных значений** — создайте файл `.env.example`:

```
# Скопируйте этот файл в .env и заполните реальными значениями
# cp .env.example .env

GITHUB_PERSONAL_ACCESS_TOKEN=
TIMEWEB_API_TOKEN=
```

Этот файл **можно и нужно** коммитить в GitHub — в нём нет реальных данных,
только названия переменных. Любой участник команды поймёт что нужно заполнить.

## Переменные для деплоя на Timeweb

При деплое на Timeweb переменные окружения задаются не в `.env`,
а прямо в панели управления:

1. Войдите на [timeweb.cloud](https://timeweb.cloud)
2. Откройте ваше приложение
3. Перейдите в **Настройки → Переменные окружения**
4. Добавьте каждую переменную: название и значение
5. Нажмите **Сохранить** — приложение перезапустится с новыми значениями

> Это правильно: на сервере переменные хранятся в панели управления,
> не в файлах — файлы могут быть доступны посторонним.

## Быстрая проверка — всё ли правильно

Попросите агента Cursor:
```
Проверь мой .env файл — правильно ли он оформлен и все ли переменные на месте
```

Или сами проверьте по чеклисту:

- [ ] Файл называется `.env` (с точкой в начале)
- [ ] Каждая переменная на отдельной строке
- [ ] Формат: `НАЗВАНИЕ=значение` (без пробелов у `=`)
- [ ] `.env` добавлен в `.gitignore`
- [ ] В `mcp.json` токены записаны как `${НАЗВАНИЕ}` или файл в `.gitignore`
- [ ] Создан `.env.example` с пустыми значениями

## Частые вопросы

**Файл .env не видно в списке файлов**
На Mac в Finder файлы с точкой скрыты. В Cursor они видны всегда.
Если не видите — нажмите `Cmd + Shift + .` в Finder чтобы показать скрытые.

**Создал .env, но агент всё равно не видит токен**
Перезапустите Cursor — он читает `.env` при запуске.

**Случайно закоммитил .env с токеном**
1. Немедленно создайте новый токен (старый уже скомпрометирован)
2. Добавьте `.env` в `.gitignore`
3. Удалите файл из истории Git — попросите агента:
   `Удали .env из истории git и из индекса`

## Структура файлов после настройки

```
мой-проект/
├── .env                 ← НЕ в GitHub (токены здесь)
├── .env.example         ← В GitHub (шаблон без значений)
├── .gitignore           ← В GitHub (включает .env)
├── .cursor/
│   ├── mcp.json         ← НЕ в GitHub (токены здесь)
│   └── mcp.example.json ← В GitHub (шаблон без токенов)
└── docs/
    └── ENV-SETUP.md     ← В GitHub (эта инструкция)
```
