# SETUP-ALL — Быстрый старт для нового участника проекта

> Этот документ — единственное что нужно прочитать в начале.
> Он проведёт вас по всем шагам в правильном порядке.
> Общее время: 30–40 минут один раз.

---

## Прежде чем начать — что вам понадобится

| Что | Где взять | Статус |
|---|---|---|
| Cursor (редактор) | [cursor.com](https://cursor.com) | Скачать и установить |
| Node.js (версия LTS) | [nodejs.org](https://nodejs.org) | Скачать и установить |
| Аккаунт GitHub | [github.com](https://github.com) | Зарегистрироваться |
| Аккаунт Timeweb Cloud | [timeweb.cloud](https://timeweb.cloud) | Зарегистрироваться |

### Как проверить что Node.js установлен

1. Откройте терминал
   - Mac: `Cmd + Пробел` → напишите «Терминал» → Enter
   - Windows: `Win + R` → напишите `cmd` → Enter
2. Напечатайте `node --version` и нажмите Enter
3. Если появилось `v20.x.x` или похожее — всё готово ✅
4. Если ошибка — установите Node.js по ссылке выше, кнопка **LTS**

## Шаг 1 — Токены и переменные окружения

> Подробная инструкция: [ENV-SETUP.md](./ENV-SETUP.md)

Токен — это «пароль-ключ» который даёт агенту доступ к сервису.
Переменная окружения — безопасное место где этот ключ хранится.

### 1.1 Получите токен GitHub

1. Войдите на [github.com](https://github.com)
2. Аватар (правый верхний угол) → **Settings**
3. Прокрутите вниз левое меню → **Developer settings**
4. **Personal access tokens → Tokens (classic)**
5. **Generate new token (classic)**
6. Note: `cursor-mcp`, срок: **No expiration**
7. Галочки: ✅ `repo` (все), ✅ `workflow`, ✅ `read:org`
8. **Generate token** → скопируйте сразу (показывается один раз!)

> Токен начинается с `ghp_...` — сохраните в заметки.

### 1.2 Получите токен Timeweb

1. Войдите на [timeweb.cloud](https://timeweb.cloud)
2. Аватар → **Настройки аккаунта → API** (или Токены)
3. **Создать токен** → название `cursor-mcp`
4. Скопируйте и сохраните

### 1.3 Создайте файл .env

1. Откройте папку проекта в Cursor
2. В левой панели: правая кнопка мыши → **New File**
3. Назовите файл: `.env` (точка в начале обязательна)
4. Напишите внутри:
   ```
   GITHUB_PERSONAL_ACCESS_TOKEN=ghp_вашТокенGitHub
   TIMEWEB_API_TOKEN=вашТокенTimeweb
   ```
5. Сохраните: `Cmd + S` (Mac) или `Ctrl + S` (Windows)

### 1.4 Защитите токены от попадания в GitHub

1. Найдите файл `.gitignore` в корне проекта (или создайте его)
2. Добавьте строки:
   ```
   .env
   .env.local
   .cursor/mcp.json
   ```
3. Сохраните

✅ **Шаг 1 готов** — токены созданы и защищены

## Шаг 2 — Подключение MCP в Cursor

> Подробная инструкция: [MCP-SETUP.md](./MCP-SETUP.md)

MCP даёт агенту Cursor «руки» — он сможет сам делать коммиты,
создавать ветки, деплоить проект по вашей команде.

### 2.1 Откройте настройки MCP

1. Откройте Cursor
2. Нажмите `Cmd + Shift + P` (Mac) или `Ctrl + Shift + P` (Windows)
3. Напечатайте `mcp` → выберите **Open MCP Config**
4. Откроется файл `mcp.json`

### 2.2 Вставьте конфигурацию

Замените всё содержимое файла на:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_вашТокенGitHub"
      }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
    }
  }
}
```

Замените `ghp_вашТокенGitHub` на реальный токен из Шага 1.

### 2.3 Перезапустите Cursor

1. Полностью закройте Cursor (не просто окно)
2. Откройте снова
3. Зайдите в **Settings → Tools & MCP**
4. Рядом с `github` и `filesystem` должны быть **зелёные точки** ✅

### 2.4 Проверьте что работает

1. Откройте чат Cursor (правая панель)
2. Убедитесь что выбран режим **Agent**
3. Напишите: `Покажи список файлов в текущей папке`
4. Агент должен ответить — значит MCP работает ✅

**Если точка красная:**
- Проверьте Node.js: в терминале `node --version`
- Если ошибка — установите Node.js и перезапустите Cursor

✅ **Шаг 2 готов** — агент подключён к инструментам

## Готово — как теперь работать каждый день

После настройки ваш рабочий процесс:

```
1. Открыли Cursor
2. Описали задачу агенту
3. Агент сделал изменения
4. Сказали: "Закоммить и запушь в main"
5. Через 2-5 минут сайт обновлён автоматически
```

### Команды которые стоит запомнить

```
"Закоммить изменения с сообщением '[что сделано]' и запушь в main"
"Покажи статус изменений"
"Создай ветку [название] для новой функции"
"Сделай pull request из [ветка] в main"
"Деплой упал, вот лог: [текст] — что делать?"
"Откатись к предыдущей рабочей версии"
```

## Что делать если что-то пошло не так

| Проблема | Решение |
|---|---|
| Зелёной точки нет в MCP | Установите Node.js, перезапустите Cursor |
| Агент не видит инструменты | Переключите режим на **Agent** в чате |
| Деплой упал (красный статус) | Скопируйте лог, вставьте агенту |
| Сайт не открывается | Проверьте переменные окружения в Timeweb |
| Случайно залили токен в GitHub | Создайте новый токен, старый аннулирован |
| Не знаете команду сборки | Спросите агента: «Какая команда сборки для моего проекта?» |
