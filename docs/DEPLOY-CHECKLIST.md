# DEPLOY-CHECKLIST.md

Универсальный чеклист перед деплоем, пригодный для любых платформ.

## Когда использовать

Использовать перед каждым релизом в staging и особенно перед production.

## Чеклист

### 1. Код и версия

- [ ] Все нужные изменения закоммичены.
- [ ] Понятно, какая ветка деплоится.
- [ ] В `CHANGELOG.md` есть краткая запись о релизе.
- [ ] Нет незакоммиченных критичных файлов.

### 2. Конфигурация

- [ ] Все обязательные переменные описаны в `ENV.md`.
- [ ] Секреты не попали в git.
- [ ] Конфигурация staging и production различается только там, где это ожидаемо.
- [ ] Домены, callback URL и API endpoints проверены.

### 3. Качество

- [ ] Проект собирается без ошибок.
- [ ] Линтер проходит или есть осознанное исключение.
- [ ] Тесты проходят или зафиксировано, почему их нет.
- [ ] Критический пользовательский сценарий проверен вручную.

### 4. Данные и миграции

- [ ] Понятно, нужны ли миграции базы данных.
- [ ] Понятно, обратимы ли миграции.
- [ ] Есть бэкап или понятный путь отката, если затронуты данные.
- [ ] Нет опасных ручных SQL-команд без подтверждения.

### 5. Инфраструктура

- [ ] Целевая среда доступна.
- [ ] Есть права на деплой.
- [ ] Логи доступны.
- [ ] Health-check endpoint известен.

### 6. После деплоя

- [ ] Известно, кто проводит smoke-тест.
- [ ] Известно, где смотреть мониторинг.
- [ ] Известно, как запускать откат.
- [ ] Владелец подтвердил production-релиз.

## Правило остановки

Если хотя бы один критичный пункт не выполнен, production-деплой запрещён.
