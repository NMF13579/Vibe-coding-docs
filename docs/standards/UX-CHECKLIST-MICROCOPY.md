# UX-CHECKLIST-MICROCOPY.md

Чек-лист микротекста (microcopy).
`[x]` — по умолчанию, `[ ]` — опционально.

## 1. Общий стиль

- [x] Короткие фразы.
- [x] Понятный язык без лишнего жаргона.
- [x] От первого лица системы или нейтрально (без путаницы).
- [ ] Согласованный tone of voice для всего продукта.

## 2. Кнопки и действия

- [x] Подписи кнопок описывают действие (“Сохранить”, “Отправить”, а не “Ок”).
- [x] Обратные действия понятны (“Отменить”, “Вернуться”).
- [ ] Тексты на кнопках согласованы across продукт.

## 3. Ошибки и предупреждения

- [x] Ошибка объясняет, что не так и что делать.
- [x] Без обвинения пользователя.
- [x] Без технического шума, если он не нужен.
- [ ] Отдельные тексты для разных типов ошибок (форма, сеть, доступ).

## 4. Пустые состояния и onboarding

- [x] Microcopy в empty states помогает начать, а не просто констатирует факт.
- [ ] Тон дружелюбный, но не назойливый.
- [ ] Для медицинских экранов — аккуратный, спокойный язык.

## 5. Медицинский контекст

- [x] Нет чрезмерно тревожных формулировок там, где ситуация не критична.
- [x] Критичные ситуации обозначены ясно и уважительно.
- [ ] Отдельные шаблоны текстов для пациента и для врача.

## 6. Правило применения

- Агент использует понятный, короткий microcopy как стандарт.
- При сомнении между “официальным” и “человеческим” текстом — выбирать более понятный, но корректный.
