# UX-CHECKLIST-HISTORY-AND-AUDIT.md

Чек-лист истории и аудита действий.
Особенно важен для медицинских, юридических и B2B-систем.
`[x]` — по умолчанию, `[!]` — обязательно, `[ ]` — опционально.

## 1. История изменений

- [x] Для ключевых сущностей есть история изменений.
- [x] Видно, что было изменено.
- [x] Видно, когда было изменено.
- [ ] Возможность отфильтровать по типу изменений.

## 2. Кто что сделал

- [!] Видно, кто внёс изменение (пользователь/роль).
- [x] Для автоматических событий указан источник (система, интеграция).
- [ ] Сводная история по пользователю.

## 3. Интерфейс истории

- [x] История читаема: упорядочена по времени.
- [x] Важные события выделены визуально.
- [ ] Возможность разворачивать подробности.
- [ ] Фильтры и поиск по истории.

## 4. Медицинский контекст

- [!] История видна там, где изменение влияет на пациента/данные.
- [!] Критичные изменения явно обозначены.
- [!] Указано, кто и когда внёс клинически значимое изменение.
- [ ] Отдельный журнал аудита для администраторов и проверок.
- [ ] Ограничение доступа к части истории по ролям.

## 5. Правило применения

- Агент добавляет историю изменений к сущностям, где важно “кто и что поменял”.
- В медицинских системах история — не опция, а стандарт.
