# UX-CHECKLIST-ROLE-BASED-UX.md

Чек-лист role-based UX.
Нужен, когда у системы несколько типов пользователей.
`[x]` — по умолчанию, `[!]` — обязательно, `[ ]` — опционально.

## 1. Определение ролей

- [!] Роли и основные типы пользователей определены.
- [x] Для каждой роли описаны основные цели и задачи.
- [ ] Зафиксированы ограничения по доступу и функциям.

## 2. Интерфейсы по ролям

- [x] Начальные экраны различаются для разных ролей (если нужно).
- [x] Меню и дашборды адаптированы под роль.
- [!] Пользователь не видит лишнего.
- [ ] Отдельные onboarding-сценарии по ролям.

## 3. Навигация по ролям

- [x] Врач/оператор/администратор видит релевантные разделы.
- [x] Не показываются пункты, к которым нет доступа (или ясно помечены).
- [ ] Быстрые ссылки на ключевые задачи для каждой роли.

## 4. Медицинский контекст

- [!] Врач видит клинический контекст и нужные инструменты.
- [!] Пациент видит упрощённый и безопасный интерфейс.
- [!] Администратор видит настройки и отчёты, но не лишние клинические детали.
- [ ] Для каждого типа пользователя есть пример основного сценария.

## 5. Правило применения

- Агент проектирует интерфейс, исходя из роли, а не “универсальной” страницы для всех.
