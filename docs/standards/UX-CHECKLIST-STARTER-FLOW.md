# UX-CHECKLIST-STARTER-FLOW.md

Правило старта беседы для выбора UX-чек-листов.

## 1. Когда запускать

Агент запускает этот сценарий:
- в начале новой проектной беседы;
- при старте нового модуля/раздела;
- если UX-чек-листы ещё не были выбраны.

## 2. Цель

Не начинать проектирование интерфейса без минимального согласования UX-основы.

## 3. Что агент спрашивает в начале

Агент задаёт короткие вопросы:

1. Какой тип проекта?
- MVP / простой кабинет
- Внутренний сервис / админка
- Медицинский / около-медицинский сервис
- Пациентский кабинет / портал
- Другое

2. Где будет использоваться интерфейс?
- Веб
- Мобильный
- Mobile-first
- Desktop-first

3. Насколько интерфейс должен быть простым?
- Минимальный
- Стандартный
- Расширенный

4. Есть ли чувствительные или медицинские данные?
- Да
- Нет

5. Нужны ли:
- сложные списки и фильтры;
- история изменений;
- уведомления;
- dashboard;
- разные роли пользователей.

## 4. Что агент делает после ответов

Агент сам предлагает набор чек-листов.

### 4.1. Базовый набор по умолчанию
- `UX-CHECKLIST-DEFAULT.md`
- `UX-CHECKLIST-ACCESSIBILITY.md`
- `UX-CHECKLIST-EMPTY-STATES.md`

### 4.2. Если пользователь просит простой интерфейс
- заменить `DEFAULT` на `MINIMAL`

### 4.3. Если проект медицинский
- добавить:
  - `UX-CHECKLIST-MEDICAL.md`
  - `UX-CHECKLIST-HISTORY-AND-AUDIT.md`
  - `UX-CHECKLIST-ROLE-BASED-UX.md`
  - `UX-CHECKLIST-NOTIFICATIONS.md`

### 4.4. Если проект mobile-first или с сильным мобильным сценарием
- добавить:
  - `UX-CHECKLIST-MOBILE.md`

### 4.5. Если много данных, списков, реестров
- добавить:
  - `UX-CHECKLIST-SEARCH-FILTERS.md`
  - `UX-CHECKLIST-TABLES-AND-BULK-ACTIONS.md`

### 4.6. Если нужен обзорный экран
- добавить:
  - `UX-CHECKLIST-DASHBOARDS.md`

## 5. Как агент формулирует предложение

Пример:

"Предлагаю такой UX-набор для старта:
- DEFAULT — базовая структура интерфейса;
- ACCESSIBILITY — доступность;
- EMPTY-STATES — пустые состояния;
- MEDICAL — безопасный медицинский UX;
- MOBILE — mobile-first паттерны;
- HISTORY-AND-AUDIT — история изменений;
- NOTIFICATIONS — уведомления.

Подтверждаете этот набор как стандарт для проекта?"

## 6. Если пользователь не хочет выбирать подробно

Агент сам предлагает рекомендованный набор и просит подтвердить его одним ответом:
- "Да, применяем"
- "Сделай минимальный вариант"
- "Без mobile"
- "Без dashboard"
- "Только базовый набор"

## 7. Если выбор не сделан

Если пользователь не ответил на выбор чек-листов, агент должен использовать:
- `UX-CHECKLIST-FALLBACK-RULES.md`

То есть перед проектированием каждого нового раздела предлагать краткий список UX-элементов для этого экрана.
