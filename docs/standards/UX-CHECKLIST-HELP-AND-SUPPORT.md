# UX-CHECKLIST-HELP-AND-SUPPORT.md

Чек-лист помощи и поддержки.
`[x]` — по умолчанию, `[ ]` — опционально.

## 1. Доступ к помощи

- [x] В интерфейсе есть ясное место, куда идти за помощью.
- [x] Помощь не спрятана глубоко.
- [ ] Быстрая подсказка на каждом ключевом экране.

## 2. Формат помощи

- [x] Короткие справки и ответы на частые вопросы.
- [ ] Отдельная страница/центр помощи.
- [ ] Контекстные подсказки в интерфейсе.
- [ ] Видео или интерактивное обучение (по запросу).

## 3. Связь с поддержкой

- [ ] Контакты поддержки или форма обратной связи.
- [ ] Возможность оставить запрос с контекстом (экран, ошибки).
- [ ] Информация, что ожидать (сроки ответа, формат).

## 4. Медицинский контекст

- [x] Ясно, к кому обращаться по клиническим вопросам, а к кому — по техническим.
- [ ] Раздел для пациента и для врача с разным уровнем деталей.
- [ ] Отдельные инструкции по критичным сценариям.

## 5. Правило применения

- Агент всегда предусматривает хотя бы базовый пункт “помощь/поддержка”.
- Расширенные центры помощи внедряются по мере роста продукта.
