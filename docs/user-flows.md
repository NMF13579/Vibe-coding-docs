# USER-FLOWS — ключевые пользовательские сценарии

> Этот файл нужен для того, чтобы агент и владелец проекта
> понимали, какие сценарии обязательно проверять перед релизом.

---

## 1. Авторизация

### 1.1. Регистрация нового пользователя
- Открыть страницу регистрации.
- Заполнить все обязательные поля.
- Отправить форму.
- Убедиться, что пользователь:
  - либо попадает в приложение,
  - либо получает понятное сообщение об ошибке.

### 1.2. Вход существующего пользователя
- Открыть страницу входа.
- Ввести корректный логин и пароль.
- Убедиться, что пользователь попадает на главный экран приложения.

### 1.3. Поведение при неверных данных
- Ввести неверный пароль.
- Убедиться, что:
  - пользователь остаётся на странице логина;
  - показывается понятное сообщение об ошибке.

---

## 2. Основной сценарий приложения

> Здесь владелец проекта ДОПИСЫВАЕТ, что является «главным действием».
> Например: создание записи, загрузка файла, оформление заказа и т.д.

### 2.1. Создание объекта
- Залогиниться.
- Открыть экран создания.
- Заполнить форму минимально необходимыми данными.
- Сохранить.
- Убедиться, что объект появился:
  - в списке;
  - на нужной странице.

### 2.2. Редактирование объекта
- Открыть существующий объект.
- Изменить одно-два поля.
- Сохранить.
- Убедиться, что изменения видны после обновления страницы.

### 2.3. Удаление / архивирование объекта
- Открыть существующий объект.
- Нажать кнопку удаления / архивации.
- Подтвердить действие, если есть диалог.
- Убедиться, что объект:
  - пропал из основного списка;
  - либо появился в архиве (если предусмотрено).

---

## 3. Навигация и доступность

### 3.1. Навигация по основным разделам
- Залогиниться.
- Перекликать все пункты основного меню.
- Убедиться, что:
  - каждая страница открывается без ошибок;
  - заголовок страницы соответствует содержимому.

### 3.2. Выход из аккаунта
- Залогиниться.
- Нажать кнопку выхода.
- Убедиться, что:
  - сессия завершена;
  - защищённые страницы более не доступны.

---

## 4. Сценарии для smoke-теста перед релизом

Перед любым релизом агент должен минимум:

- [ ] Проверить регистрацию нового пользователя (если доступна).
- [ ] Проверить вход существующего пользователя.
- [ ] Выполнить основной сценарий (создание объекта).
- [ ] Открыть 2–3 ключевых экрана приложения.
- [ ] Проверить выход из аккаунта.

---

## 5. Как дополнять этот файл

Владелец проекта может добавить новые сценарии:

- Новый раздел: `## N. Название сценария`.
- Внутри — подпункты шаг за шагом.
- Каждый новый сценарий должен быть понятен без контекста.
