Who is this for?

Pick your role to see how the framework helps you work with AI agents step by step.

How it works

Four stages keep your product story in one place so agents do not start from zero.

Install the plugin

One command (or a short settings step) adds interviews, specs, and checklists to your workspace.

Describe your project

Interview stage captures goals, users, and constraints in a single living document.

Agent builds with context

Development follows the same file so changes stay aligned with what you already decided.

Deploy with memory intact

Handoff notes and status files help the next session pick up without re-explaining everything.

Install

Choose your tool and copy the snippet. Everything else lives in the repo README.

/plugin install vibe-coding-docs@https://github.com/NMF13579/Vibe-coding-docs.git

Run inside Claude Code. The plugin pulls the latest framework files into your project.

In Cursor: open Settings → Rules and add a rule that points to the project CLAUDE.md file (or paste its path). That file is the entry point for the same workflow.

Medical Professional

Ship safe, useful clinical helpers by steering the agent with clear product facts—not code.

  1. Run the interview. Capture the clinical scenario, who uses the tool, and what must never happen.
  2. Keep one source of truth. Let updates go into the project doc so every session sees the same story.
  3. Review outputs, do not debug. Ask the agent to explain screens and flows in plain language before you approve.
  4. Involve compliance early. Note consent, data handling, and jurisdiction in the locked sections when your team is ready.

Designer

Move from pixels to a working prototype while preserving UX intent across agent handoffs.

  1. Attach references. Export key frames and link them in the discovery notes the template asks for.
  2. Define states, not components. Describe empty, loading, error, and success so build prompts stay concrete.
  3. Iterate in tight loops. Ask for a screen, react with screenshots or bullet feedback, then update the spec.
  4. Use the framework gates. Let “plan → review → confirm” stop surprise redesigns mid-build.

Manager

Coordinate vendors or internal builders without losing decisions between AI sessions.

  1. Start with outcomes. Fill success metrics and deadlines in the interview so agents optimize for the business case.
  2. Assign ownership. Record who approves copy, brand, and releases inside the project file.
  3. Track scope. When ideas grow, log “deferred” decisions instead of silent scope creep.
  4. Read handoff notes. Open the status doc before each session so you see what changed last time.

Developer

Give vibe-coding agents structure: contracts, layers, and review steps that scale.

  1. Install once per repo. Keep `CLAUDE.md`, layers, and stage BOOT files at the paths the template expects.
  2. Respect locks. Teach collaborators which `PROJECT.md` sections need explicit approval before edits.
  3. Automate checks. Point CI at the same smoke tests referenced in the dev stage docs.
  4. Document ADRs. When stack choices change, append to decisions so agents do not revert them.
