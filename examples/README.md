# AgentOS Example Scenarios

## Purpose

These examples show how AgentOS is intended to be used in practical AI-assisted coding workflows.
They are documentation scenarios, not executable fixtures.

## Scenarios

| Scenario | Purpose |
|---|---|
| scenario-01-new-feature.md | From feature idea to reviewed draft contract |
| scenario-02-bugfix.md | Narrow bugfix with acceptance criteria and rollback notes |
| scenario-03-refactor.md | Scope-controlled behavior-preserving refactor |
| scenario-04-validation-only.md | Repository health validation without task execution |

## Safety Notes

- Examples do not execute tasks.
- Examples do not replace `tasks/active-task.md`.
- Examples do not move queue items.
- Examples do not approve release.
- Examples do not bypass human checkpoints.
