# Архитектура Claude Code (docs-first, один источник истины)

> 👤 **Этот документ для владельца, не для агента.**  
> Здесь собраны фразы, которые нужно написать агенту на каждом этапе. Правила поведения агента — в [`CLAUDE.md`](./CLAUDE.md).

Кратко: как устроен **поток только для Claude Code / Claude Projects** в этом репозитории.

## Что это

- **Один живой spec продукта:** `project/PROJECT.md` (Discovery, UX, Specs, Deploy, Decisions).
- **Один этап — один контекст:** папки `stages/01-interview` … `04-deploy`, в каждой свой `BOOT.md` (что читать, куда писать, smoke-test).
- **Общие правила для всех этапов:** `shared/` (сжатая выжимка + ссылки на полные `LAYER-*` файлы).

## Как открыть новый проект

1. Скопируй или сгенерируй репозиторий из шаблона.
2. Открой в **Claude Code**; агент читает **`CLAUDE.md`**.
3. Начни с **`stages/01-interview/BOOT.md`** (или скажи агенту прочитать его).

## Как запускать этапы

| Этап | Команда владельцу (пример) |
|------|----------------------------|
| Интервью | «Прочитай `stages/01-interview/BOOT.md` и начни этап интервью» |
| UX | «Прочитай `stages/02-ux/BOOT.md` и предложи UX-план» |
| Разработка | «Прочитай `stages/03-dev/BOOT.md`, работай только по SPECS» |
| Деплой | «Прочитай `stages/04-deploy/BOOT.md` и план выката» |

Общие фразы:

- «Сначала план на русском, потом жду моего „да"» — включает цикл **Plan → Review → Confirm → Execute**.
- «Работай только по разделу SPECS в `project/PROJECT.md`, не меняй Discovery и UX» — границы секций.

## Если что-то сломалось

1. Проверь, что агент открыл **`BOOT.md` нужного этапа**, а не весь репозиторий подряд.
2. Открой `project/PROJECT.md`: нет ли правок в чужой секции или в `LOCKED:true`.
3. Восстановление контекста шаблона: `HANDOFF.md`, `LAYER-3/project-status.md`, `LAYER-1/context-recovery.md`.
4. Перед релизом: `CHECKLIST.md`.

## Связь с canonical modules

Полная runtime-навигация идёт через `llms.txt`, `ROUTES-REGISTRY.md` и пять canonical modules.
`shared/` и `stages/*/*.md` остаются вспомогательными заметками и не задают правила выполнения.

## Архив

Закрытые большие куски при росте `PROJECT.md` можно выносить в архивную папку проекта; краткое резюме остаётся в основном файле.
