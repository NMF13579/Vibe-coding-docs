# FEATURE: Авторизация пользователей

## Зачем (Why)
Пользователи должны иметь личный кабинет чтобы данные 
сохранялись между сессиями и были приватны.

## Что именно (What)
Пользователь может зарегистрироваться по email и паролю,
войти в систему, выйти. После входа видит только свои данные.

## Границы (Scope)
### Входит в фичу
- Регистрация по email + пароль
- Вход / выход
- Защита роутов — неавторизованный видит только /login

### НЕ входит в фичу
- OAuth (Google, GitHub) — следующая итерация
- Восстановление пароля — отдельная фича
- Роли и права — отдельная фича

## Acceptance Criteria фичи
- [ ] Незарегистрированный пользователь не может открыть /dashboard
- [ ] После регистрации пользователь автоматически входит в систему
- [ ] После выхода сессия уничтожена, /dashboard недоступен
- [ ] Неверный пароль показывает понятное сообщение об ошибке
