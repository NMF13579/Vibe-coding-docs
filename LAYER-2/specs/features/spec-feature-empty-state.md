# FEATURE: Empty states для всех экранов

## Зачем (Why)
Когда данных нет — пользователь видит пустой экран и не понимает
что делать. Это создаёт ощущение сломанного продукта.

## Что именно (What)
Каждый экран с данными показывает дружелюбное сообщение 
и подсказку-действие когда список пуст.

## Границы (Scope)
### Входит в фичу
- Empty state для каждого списка в приложении
- Иконка + заголовок + подсказка что делать
- Кнопка действия где применимо

### НЕ входит в фичу
- Анимации — отдельная задача
- Onboarding flow — отдельная фича

## Acceptance Criteria фичи
- [ ] Ни один экран с данными не показывает просто пустоту
- [ ] Каждый empty state содержит подсказку следующего действия
- [ ] Empty state выглядит одинаково на всех экранах (единый стиль)

## Зависимости
- Требует: базовые экраны готовы
