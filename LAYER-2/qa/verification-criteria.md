# VERIFICATION CRITERIA — стандарт доказательства готовности

> Задача закрыта когда агент ДОКАЗАЛ что AC выполнены,
> а не когда написал код.
>
> Медицинская аналогия: не «выписали рецепт» — а «пациент выздоровел».

---

## Принцип

Каждый пункт Acceptance Criteria требует одного из доказательств:

| Тип AC | Допустимое доказательство |
|--------|--------------------------|
| UI отображается | Скриншот / описание что видно в браузере |
| Нет ошибок TypeScript | Вывод `tsc --noEmit` без ошибок |
| Функция работает | Результат вызова с конкретными данными |
| Данные сохраняются | До/после: показать что было и что стало |
| Тест проходит | Вывод `npm test` с зелёными результатами |
| Роут защищён | Попытка доступа без авторизации → редирект |

---

## Формат доказательства (агент заполняет в задаче)

После завершения работы агент добавляет в раздел 
«Отчёт о выполнении» блок:

````
```
## Доказательства (Verification Evidence)

AC: [текст пункта из Acceptance Criteria]
Доказательство: [что именно проверено и результат]
Метод: [как проверено: tsc / консоль / браузер / тест]

AC: [следующий пункт]
Доказательство: ...
Метод: ...
```
````

---

## Уровни доказательств

### 🟢 Сильное доказательство (принимается без вопросов)
- Вывод команды без ошибок (tsc, npm test, lint)
- Конкретный результат вызова функции с данными
- Описание поведения с шагами воспроизведения

### 🟡 Слабое доказательство (принимается с оговоркой)
- «Код выглядит правильно»
- «Должно работать по логике»
- «Аналогично предыдущей реализации»

### 🔴 Не является доказательством
- «Сделано»
- «Готово»
- «Реализовано согласно задаче»

> ⚠️ Слабое доказательство фиксируется в задаче как технический долг.
> Красное доказательство = задача не закрыта.

---

## Исключения

| Ситуация | Действие |
|----------|----------|
| AC требует действия пользователя (нажать кнопку) | Агент описывает шаги и просит владельца подтвердить |
| AC невозможно проверить автоматически | Явно отметить: «Требует ручной проверки владельцем» |
| Среда не позволяет запустить тест | Зафиксировать причину, предложить альтернативу |

---

## Связь с другими протоколами

- AC берутся из `tasks/TASK-XXX.md → ## Acceptance Criteria`
- Доказательства пишутся в `## Отчёт о выполнении` той же задачи
- TASK-REVIEW-PROTOCOL Блок 1 проверяет наличие доказательств
- Без доказательств → задача не переходит в статус ✅ Завершена
