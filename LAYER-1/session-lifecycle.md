# Session lifecycle — жизненный цикл сессии

Единое описание старта, завершения задачи, напоминаний и триггеров (раньше дублировалось в `CLAUDE.md`, Copilot, Cursor `00-core.mdc`).

---

## Старт сессии (onSessionStart)

1. Прочитать `LAYER-3/STATE.md` (если есть) — Project / Session / Task, guards.
2. Прочитать `HANDOFF.md` (Terminal Snapshot и при необходимости Session History) и `LAYER-3/project-status.md`.
3. Прочитать `LAYER-3/lessons.md`, если файл существует.
4. Если одна и та же задача снова выглядит незакрытой — предупредить: «⚠️ Задача [название] идёт уже несколько сессий. Разбить на части или продолжить?»
5. Кратко (3–5 строк): состояние, что было в прошлый раз, 1–3 варианта следующего шага.
6. Дождаться выбора владельца — не писать код до явного согласования следующего шага (см. `LAYER-1/workflow.md`, «Молчание ≠ согласие»).

**Субагенты в Claude Code** (audit-agent, feature-advisor-agent, session-guard-agent) в других средах не вызываются — использовать документы: `LAYER-1/audit.md`, `LAYER-1/feature-radar.md` + `LAYER-3/features.md`, триггеры сохранения ниже.

---

## После завершённой задачи (onTaskComplete)

1. Обновить `HANDOFF.md` (Terminal Snapshot; при необходимости строка в Session History).
2. Обновить `LAYER-3/STATE.md` (все три домена) — см. `LAYER-1/agent-rules.md`, HANDOFF PROTOCOL.
3. Обновить `LAYER-3/project-status.md`.
4. При необходимости — `LAYER-3/deferred-decisions.md`, `LAYER-3/fixes.md`, `LAYER-3/security.md` и т.д.
5. Сообщить: «Контекст сохранён ✅».

---

## Напоминание о сохранении (onSaveReminder)

Напомнить («⚠️ Контекст не сохранялся. Сохранить сейчас?»), если:

- с последнего обновления `HANDOFF.md` сделано **10+** правок файлов, **или**
- выполнен план из **3+** шагов без обновления `HANDOFF.md`.

---

## Session guard (~30 минут активной работы)

Перенесено из `.claude/agents/session-guard-agent.md` (Claude Code).

- Следить, обновлялся ли `HANDOFF.md` примерно за последние **30 минут** активной работы.
- Если **нет** — напомнить:

  > ⚠️ **Контекст не сохранялся ~30 минут.**  
  > Сохранить сейчас?

- Не прерывать мысль пользователя — напоминать, когда уместна пауза.
- Если пользователь попросил сохранить — обновить `HANDOFF.md` и `LAYER-3/project-status.md` сразу.

---

## Аудит и здоровье (onAuditRequest)

Триггеры: «проверь проект», «аудит», «здоровье».

- Следовать `LAYER-1/audit.md` (протокол и раздел **«Чеклист аудита»**).
- Для короткой оценки готовности — блок **«Фокус: аудит здоровья / готовности»** в начале `audit.md`.

---

## Идеи и «что дальше» (onFeatureRequest)

1. `LAYER-1/feature-radar.md` и `LAYER-3/features.md`.
2. Предложить **три** варианта с кратким обоснованием.

### Детали ответа (эквивалент feature-advisor-agent, Claude Code)

Перенесено из `.claude/agents/feature-advisor-agent.md`.

- Не больше **3–5** улучшений за один ответ.
- Для каждого: что это, зачем сейчас, какой эффект; с чего начать.
- Не тащить enterprise-сложность слишком рано; простой язык; ощущение ясности и контроля.

---

## Принципы работы в репозитории (кратко)

- Опираться на существующие документы, а не на догадки.
- Простой язык; **один вопрос за раз**, если неясно.
- Практичные, приоритизированные предложения; без лишней сложности.
