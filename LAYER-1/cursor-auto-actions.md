# Cursor — Auto-Actions (псевдо-хуки)

Таблица перенесена из устаревшего `.cursor/CLAUDE-WORKFLOW.md` (логика не привязана к формату IDE-файла).

Следующие действия выполняются **по смыслу триггера** (когда агент фиксирует событие):

| Триггер | Автоматическое действие |
|---|---|
| Задача помечена как выполненная (Done / ✅) | Обновить статус в `HANDOFF.md` + `LAYER-3/project-status.md` |
| Создан или изменён файл в `LAYER-2/specs/features/` | Проверить: заполнен ли `_template.md` полностью (все секции, статус не Draft) |
| Фича затрагивает данные пациентов, диагнозы, МИС | Предложить Medical Review: «Эта фича требует Medical Reviewer — запустить?» |
| Завершена задача типа TASK-XXX | Спросить: «Обновить CHANGELOG.md?» |
| Обнаружена ошибка / баг в процессе работы | Зафиксировать в `LAYER-3/fixes.md` до исправления, не править молча |
| Scope расширяется за пределы текущей задачи | Остановиться: «⚠️ Это выходит за скоуп TASK-XXX. Согласовать расширение?» |
| Завершена сессия (последнее сообщение в потоке) | Напомнить: «Сохранить контекст в HANDOFF.md перед закрытием?» |

Этапы `stages/*`: см. [`stage-routing.md`](./stage-routing.md) (не использовать устаревшие пути `stages/01-interview/` из старых заметок).
