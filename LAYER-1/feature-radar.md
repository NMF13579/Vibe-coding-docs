# FEATURE-RADAR — какие функции предложить на текущем этапе

> Этот файл нужен чтобы агент проактивно подсказывал полезные функции, даже если пользователь не знает как их назвать.

## Этап: идея

Можно предложить:

- Описать главного пользователя.
- Описать одну самую важную проблему.
- Сузить MVP до 1–3 ключевых функций.

## Этап: MVP в сборке

Можно предложить:

- Простую авторизацию.
- Пустые состояния экранов.
- Базовую обработку ошибок.
- Роли пользователей, если в проекте больше одного типа пользователей.

## Этап: первые пользователи

Можно предложить:

- Обратную связь внутри продукта.
- Аналитику поведения пользователей.
- Логирование ошибок.
- Бэкап данных.
- Политику конфиденциальности и базовые юридические страницы.

## Этап: развитие

Можно предложить:

- Улучшение скорости.
- Онбординг новых пользователей.
- Автоматизацию рутинных действий.
- Более тонкие роли и права доступа.
- Панель администратора, если это уместно.

## Как отвечать пользователю

- Предлагать не больше 3–5 функций за раз.
- Для каждой функции объяснять зачем она нужна именно сейчас.
- Если проект ранний — не тащить enterprise-функции слишком рано.
