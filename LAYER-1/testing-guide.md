# TESTING-GUIDE — как проверять что код работает

> ⚠️ Deprecated as a primary route.
> Canonical successor: `quality/MAIN.md`.
> This document remains available as legacy/reference material during transition.

> Агент читает этот файл ПЕРЕД объявлением задачи выполненной.
> Владелец читает этот файл когда нужно сообщить об ошибке.

---

## Правило агента: задача не закрыта без проверки

После написания кода агент обязан:
1. Назвать конкретные шаги для ручной проверки
2. Указать что должно произойти (ожидаемый результат)
3. Указать что НЕ должно произойти (признаки ошибки)
4. Дождаться подтверждения от владельца

---

## Шаблон: как сообщить об ошибке

Когда что-то не работает — скопируй и заполни:
🔴 ОШИБКА

Что я делал: [опиши действие, например «нажал кнопку Сохранить»]
Что ожидал: [что должно было произойти]
Что произошло: [что произошло на самом деле]
Текст ошибки (если есть): [скопируй текст или сделай скриншот]
Когда началось: [после чего появилось — после обновления, всегда было, впервые]

text

## Уровни серьёзности ошибок

| Уровень | Признак | Что делать |
|---------|---------|-----------|
| 🔴 Критическая | Приложение не запускается / данные теряются / авторизация сломана | Стоп. Сразу сообщить агенту. Не делать ничего другого |
| 🟡 Важная | Функция не работает, но остальное ок | Закончить текущий шаг, сообщить агенту |
| 🟢 Косметическая | Выглядит не так / текст неправильный | Записать в список, сообщить после сессии |

## Чеклист после каждой задачи

Агент проверяет сам (автоматически), владелец проверяет вручную:

### Базовая проверка (все задачи)
- [ ] Приложение открывается без ошибок
- [ ] Новая функция работает по шагам из задачи
- [ ] Старые функции не сломались (проверить 2-3 ключевых)
- [ ] В браузере нет красных ошибок (открыть DevTools → Console)

### Проверка форм и ввода данных
- [ ] Форма сохраняет данные
- [ ] Форма показывает ошибку при неправильном вводе
- [ ] Форма не позволяет отправить пустые обязательные поля

### Проверка авторизации
- [ ] Неавторизованный пользователь не видит чужие данные
- [ ] После выхода из аккаунта — перенаправление на вход

### Проверка базы данных
- [ ] Данные сохраняются и видны после перезагрузки страницы
- [ ] Удалённые данные действительно удалены

## Как описать ошибку агенту эффективно

**Плохо:** «Не работает кнопка»

**Хорошо:** «Нажал кнопку Сохранить на странице профиля — ничего не произошло,
данные не сохранились. В консоли браузера красная ошибка: 403 Forbidden»

---

## Что агент делает с ошибкой

1. Классифицирует по уровню серьёзности
2. Воспроизводит по описанию
3. Называет причину простым языком
4. Предлагает конкретное исправление
5. После исправления — повторяет чеклист проверки
