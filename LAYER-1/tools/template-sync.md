# TEMPLATE-SYNC-GUIDE

# TEMPLATE-SYNC-GUIDE — руководство по синхронизации шаблона

> Для владельцев проектов: как получить обновления из Vibe-coding-docs без потери своих файлов.

---

## Что делает инструмент

`template-sync.js` сравнивает ваш проект с репозиторием-шаблоном и **копирует только те файлы, которых у вас ещё нет**.

Существующие файлы **не трогаются** — ваши изменения сохраняются.

---

## Требования

- Node.js 14+ (проверить: `node --version`)
- Git (для коммита изменений, опционально)

---

## Быстрый старт

```bash
# 1. Сухой запуск — показать что будет добавлено (ничего не меняет)
node template-sync.js /path/to/vibe-coding-docs . --dry-run

# 2. Проверить список файлов, затем применить
node template-sync.js /path/to/vibe-coding-docs .

# 3. Закоммить результат
git add .
git commit -m "sync: добавлены новые файлы из шаблона"
```

---

## Сценарии использования

### Сценарий 1 — Новый проект из шаблона

Шаблон даёт стартовую структуру, вы работаете поверх неё:

```bash
mkdir my-project && cd my-project
git init
node /path/to/vibe-coding-docs/LAYER-1/tools/template-sync.js /path/to/vibe-coding-docs .
```

Результат: все шаблонные файлы скопированы в ваш проект.

---

### Сценарий 2 — Обновление существующего проекта

Шаблон обновился и появились новые документы:

```bash
cd my-project

# Посмотреть что появилось в шаблоне
node template-sync.js /path/to/vibe-coding-docs . --dry-run

# Применить (только новые файлы — существующие не трогаются)
node template-sync.js /path/to/vibe-coding-docs .
```

---

### Сценарий 3 — Проверка перед обновлением

Если не уверены что именно изменится:

```bash
# Шаг 1: сухой запуск
node template-sync.js ../vibe-docs . --dry-run

# Шаг 2: просмотреть отчёт (создаётся автоматически при dry-run)
# Отчёт выводится в консоль

# Шаг 3: если всё ок — применить
node template-sync.js ../vibe-docs .
```

---

### Сценарий 4 — Откат если что-то пошло не так

```bash
# Посмотреть что изменилось
git status
git diff

# Откатить все изменения последнего коммита
git reset --hard HEAD~1

# Или только добавленные файлы
git clean -fd
```

---

## Что копируется, что нет

### Копируется (только если файла нет в проекте)

- `LAYER-1/*.md`, `LAYER-2/**/*.md` — документация шаблона
- `LAYER-3/*.md` — файлы памяти агента
- `tasks/` — шаблоны задач
- `CLAUDE.md`, `HANDOFF.md`, `llms.txt`, `LAYER-1/workflow.md` — файлы управления
- `LAYER-1/system-prompt.md`, `llms.txt` — конфигурация AI

### Никогда не копируется

- `README.md` — у каждого проекта свой
- `CHANGELOG.md` — история специфична для проекта
- `template-sync.js` — сам инструмент
- `template-sync-report.md` — отчёты

### Игнорируется всегда

- `.git/`, `node_modules/`
- `.env`, `.env.*` — секреты
- `.DS_Store` — системные файлы

---

## Интеграция в package.json

```json
{
  "scripts": {
    "sync:dry": "node template-sync.js ../vibe-coding-docs . --dry-run",
    "sync":     "node template-sync.js ../vibe-coding-docs .",
    "sync:report": "cat template-sync-report.md"
  }
}
```

Использование:

```bash
npm run sync:dry   # посмотреть что изменится
npm run sync       # применить
```

---

## Отчёт синхронизации

После каждого запуска (без `--dry-run`) создаётся файл `template-sync-report.md` с:

- датой и режимом запуска
- списком скопированных файлов
- списком пропущенных файлов (уже существуют)
- списком ошибок (если были)

Добавьте `template-sync-report.md` в `.gitignore` если не хотите хранить его в репозитории.

---

## Частые вопросы

**Q: Скрипт перезапишет мои изменения в CLAUDE.md?**
A: Нет. Если файл уже есть в проекте — он не трогается.

**Q: Как добавить в проект только один конкретный файл из шаблона?**
A: Просто скопируйте его вручную: `cp /path/to/vibe-docs/LAYER-1/workflow.md ./LAYER-1/workflow.md`

**Q: Нужно ли запускать синхронизацию регулярно?**
A: Только когда в шаблоне появляются новые документы. Агент подскажет если заметит нужный файл отсутствующим.

**Q: Что если я хочу обновить уже существующий файл из шаблона?**
A: Скрипт это не делает намеренно. Сравните файлы вручную: `diff ./LAYER-1/workflow.md /path/to/vibe-docs/LAYER-1/workflow.md`

# TEMPLATE-SYNC-INTEGRATION

# TEMPLATE-SYNC-INTEGRATION — интеграция синхронизации для агентов

> Инструкция для ИИ-агента: когда и как использовать template-sync.js.

---

## Когда запускать синхронизацию

Агент предлагает синхронизацию в четырёх случаях:

| Ситуация | Действие агента |
|----------|----------------|
| Владелец пишет «синхронизируй шаблон» / «обнови документы» | Запустить dry-run, показать список, ждать подтверждения |
| При диагностике (AGENT-BOOTSTRAP) обнаружены отсутствующие файлы | Предложить синхронизацию для восполнения пробелов |
| Повышение уровня (Level 0 → 1 или 1 → 2) | Синхронизировать перед созданием новых документов |
| Владелец добавляет новый проект на основе шаблона | Запустить синхронизацию как первый шаг |

---

## Протокол синхронизации (пошагово)

### Шаг 1 — Уточнить путь к шаблону

```
«Где лежит репозиторий Vibe-coding-docs?
Например: ../vibe-coding-docs или /Users/name/vibe-coding-docs»
```

Если путь уже известен из контекста — пропустить этот шаг.

### Шаг 2 — Сухой запуск

```bash
node template-sync.js <path-to-vibe-docs> . --dry-run
```

### Шаг 3 — Показать владельцу список

```
Вот что будет добавлено в проект:

+ LAYER-1/scope-guard.md
+ LAYER-1/anti-patterns.md
+ LAYER-3/deferred-decisions.md

Файлы которые уже есть — не трогаются.
Добавить?
```

### Шаг 4 — Применить после подтверждения

```bash
node template-sync.js <path-to-vibe-docs> .
```

### Шаг 5 — Закоммить

```bash
git add .
git commit -m "sync: добавлены новые файлы из шаблона vibe-coding-docs"
```

### Шаг 6 — Обновить HANDOFF.md

Добавить запись: «Синхронизированы новые файлы из шаблона: [список]»

---

## Интеграция в CLAUDE.md проекта

Добавить раздел в CLAUDE.md проекта владельца:

```markdown
## Синхронизация шаблона

Когда владелец пишет: «Синхронизируй шаблон» или «Обнови документы»

1. Уточнить путь к vibe-coding-docs (если не известен)
2. Сухой запуск: `node template-sync.js <path> . --dry-run`
3. Показать список файлов и спросить подтверждение
4. После «да»: `node template-sync.js <path> .`
5. Закоммить: `git commit -m "sync: новые файлы из шаблона"`
6. Обновить HANDOFF.md
```

---

## Интеграция в AGENT-BOOTSTRAP

При диагностике: если обнаружены отсутствующие документы из списка критичных — предложить синхронизацию вместо создания файлов вручную:

```
Обнаружены отсутствующие файлы:
- LAYER-1/scope-guard.md
- LAYER-1/anti-patterns.md

Вместо создания вручную можно синхронизировать из шаблона.
Путь к vibe-coding-docs?
```

---

## Интеграция в протокол повышения уровня (llms.txt / LAYER-1/workflow.md)

При переходе Level 0 → 1 или Level 1 → 2 — синхронизация как первый шаг:

```
Переходим на Уровень 1. Сначала добавим недостающие документы из шаблона.

node template-sync.js <path> . --dry-run
```

Это быстрее и надёжнее чем создавать каждый файл вручную.

---

## Важные правила для агента

- **Никогда не запускать без dry-run** — сначала показать владельцу что изменится
- **Молчание ≠ согласие** — ждать явного подтверждения перед применением
- **Не предлагать синхронизацию** если владелец редактировал шаблонные файлы — есть риск путаницы
- **Всегда коммитить** после синхронизации, чтобы изменения можно было откатить
- **Уточнять путь** к шаблону если он не указан в CLAUDE.md или HANDOFF.md

---

## Поведение при ошибках

| Ошибка | Действие агента |
|--------|----------------|
| Путь к шаблону не найден | Спросить владельца путь |
| Node.js не установлен | Предложить установить: `node --version` |
| Ошибка копирования файла | Сообщить, предложить скопировать вручную |
| Владелец отменил после dry-run | Ничего не делать, подтвердить отмену |
