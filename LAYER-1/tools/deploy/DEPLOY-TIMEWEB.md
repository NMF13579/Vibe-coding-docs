# DEPLOY-TIMEWEB — деплой на Timeweb Cloud (App Platform)

> Пошаговая выкладка через **Timeweb Cloud** при связке с **GitHub** и автосборкой.
> Универсальные правила релиза (staging → production, откат): [`LAYER-1/deploy-guide.md`](../../deploy-guide.md).
> Приоритет платформы над общим текстом — см. [`timeweb-cloud.md`](./timeweb-cloud.md).

Переменные окружения и токены: [ENV-SETUP.md](./ENV-SETUP.md).

---

## 1. Убедитесь, что проект в GitHub

В терминале Cursor:

```
git remote -v
```

Должен появиться адрес репозитория на github.com.

Если нет — скажите агенту:

```
Создай репозиторий на GitHub и подключи к этому проекту
```

---

## 2. Создайте приложение в Timeweb

1. Войдите на [timeweb.cloud](https://timeweb.cloud)
2. Найдите **App Platform** в левом меню
3. Нажмите **Создать** (Frontend) или **Добавить** (Backend)
4. Выберите тип проекта
5. Нажмите **Подключить GitHub** → авторизуйте Timeweb
6. Выберите репозиторий и ветку `main`
7. Укажите команду сборки (если не знаете — спросите агента):

   ```
   Какая команда сборки нужна для моего проекта?
   ```

8. Нажмите **Заказать** (Frontend) или **Запустить деплой** (Backend)

---

## 3. Добавьте переменные окружения в Timeweb

1. Откройте созданное приложение
2. **Настройки → Переменные окружения**
3. Добавьте переменные из вашего `.env` (имена и значения — как в проекте)
4. Сохраните — приложение перезапустится само

Концепция переменных: [`ENV-SETUP.md`](./ENV-SETUP.md) и [`LAYER-1/deploy-guide.md`](../../deploy-guide.md).

---

## 4. Проверьте, что сайт работает

1. В приложении Timeweb найдите технический домен:
   - Frontend: раздел **Настройки**
   - Backend: раздел **Дашборд**
2. Откройте домен в браузере — должен открываться ваш сервис

---

## После настройки

Повседневный цикл: коммит и push в отслеживаемую ветку → Timeweb пересобирает приложение (обычно 2–5 минут). Подробнее — раздел **«Готово — как теперь работать»** в [SETUP-ALL.md](./SETUP-ALL.md).
