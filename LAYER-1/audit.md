# AUDIT-FULL — полный аудит проекта перед релизом или передачей

---

## НУЛЕВОЙ ШАГ — Уточняю формат

Прежде чем начать — агент задаёт один вопрос:

> «Быстрая проверка (5–10 минут) или полный аудит перед релизом / передачей проекта (30 минут)?»

- Быстрая → агент читает `LAYER-1/audit-checklist.md` и делает короткую проверку.
- Полный → агент продолжает по шагам ниже.

⚠️ Без ответа — не двигаться дальше. Один вопрос, два варианта.

---

## ШАГ 1 — Читаю контекст проекта

Читаю эти файлы по порядку:

1. `HANDOFF.md` — текущее состояние
2. `LAYER-3/project-status.md` — статус всех задач
3. `LAYER-3/lessons.md` — накопленные уроки
4. `LAYER-3/fixes.md` — известные проблемы
5. `llms.txt` — маршруты и структура

После чтения — перехожу к Шагу 2. Молчу.

---

## ШАГ 2 — Проверяю структуру проекта

Проверяю каждый файл. Для каждого отсутствующего критичного файла — останавливаюсь и спрашиваю владельца:

> «Файл [название] не найден. Он нужен для [объяснение зачем простыми словами]. Создать его сейчас или продолжить аудит и вернуться позже?»

Жду ответа. После «продолжить» — отмечаю 🔴 и иду дальше.

### Критичные файлы

| Файл | Зачем нужен |
|------|-------------|
| `CLAUDE.md` | Главные правила для агента — без него агент работает вслепую |
| `HANDOFF.md` | Память проекта — где остановились, что дальше |
| `llms.txt` | Карта навигации — агент знает куда смотреть |
| `LAYER-1/workflow.md` | Конвейер работы — как движется проект |
| `LAYER-1/scope-guard.md` | Защита от расползания задач |
| `LAYER-1/agent-bootstrap.md` | Стартовая диагностика при каждой сессии |
| `LAYER-1/agent-contract.md` | Правила поведения агента |
| `LAYER-3/project-status.md` | Текущий статус всех задач |

### Продуктовые документы

- [ ] `LAYER-2/discovery/` — заполнены или пустые шаблоны?
- [ ] `LAYER-2/specs/` — есть ли архитектура и ТЗ?
- [ ] `LAYER-2/ux/` — есть ли описание экранов?
- [ ] `LAYER-2/qa/` — есть ли подход к тестированию?

### Что должно отсутствовать

- [ ] Папка `docs/` — если есть: «Есть старая папка docs/ — она дублирует документацию. Лучше удалить»
- [ ] Дублирующиеся файлы с похожими названиями

После проверки — перехожу к Шагу 3. Молчу.

## ШАГ 3 — Аудит по шести направлениям

### 🔵 Направление 1 — Продукт

- Понятна ли цель проекта из `HANDOFF.md`?
- Для кого он делается?
- Описан ли текущий этап?
- Не вышел ли проект за рамки изначального плана?

### 🔵 Направление 2 — Архитектура

- Описан ли технический стек в `LAYER-2/specs/`?
- Нет ли противоречий между документацией и реальным кодом?
- Актуальны ли зависимости и библиотеки?

### 🔵 Направление 3 — Процесс

- Обновлён ли `HANDOFF.md` (когда последний раз)?
- Фиксируются ли уроки в `LAYER-3/lessons.md`?
- Есть ли задачи без статуса или «подвисшие»?

### 🔵 Направление 4 — Качество

- Есть ли подход к тестированию?
- Есть ли незакрытые баги в `LAYER-3/fixes.md`?
- Проверялся ли пользовательский интерфейс по чеклистам?

### 🔵 Направление 5 — Безопасность

- Где хранятся чувствительные данные (ключи, токены, пароли)?
- Есть ли правила доступа?
- Есть ли явные риски при публикации?

### 🔵 Направление 6 — Навигация агента

- Все ли пути в `llms.txt` ведут к существующим файлам?
- Нет ли ссылок на несуществующие документы внутри `LAYER-1/`?
- Актуален ли `CHANGELOG.md`?

|---|---|---|

> «Нашёл [N] проблем которые лучше исправить до релиза. Разобраться с ними прямо сейчас — или сначала посмотришь полный отчёт?»

- «Сейчас» → аудит приостанавливается, агент устраняет проблемы. После — возвращается к Шагу 5.
- «Полный отчёт» → агент продолжает к Шагу 5.

## ШАГ 6 — Предлагаю сохранить итоги аудита

⚠️ Агент ничего не записывает без явного «да». Показывает изменения простыми словами — не кодом и не разметкой.

Агент пишет владельцу:

> Хочу сохранить итоги аудита. Вот что изменится:
>
> — В журнале проекта: добавлю запись что аудит пройден [дата], главный вывод — [одна фраза], следующий шаг — [что].
> — В списке задач: [конкретно что изменится].
> — В уроках проекта: [что стоит запомнить].
> — В списке проблем: [что зафиксирую].
>
> Записать?

- Владелец говорит «да» → агент вносит изменения и пишет: **Аудит завершён ✅ Контекст сохранён**
- Владелец говорит «нет» или вносит правки → агент принимает, показывает обновлённый список и ждёт «да».
- Владелец молчит → агент ждёт. Ничего не записывает.

## Правила для агента

- Читай файлы реально — не угадывай содержимое
- Нулевой шаг обязателен — без ответа владельца не начинать
- Если критичный файл отсутствует — останавливайся, объясняй зачем он нужен, жди выбора
- Если есть 🔴 после Шага 4 — спроси: починить сейчас или сначала полный отчёт
- Шаг 5 пишется как письмо человеку — не технический документ
- Шаг 6 — только после явного «да». Показывать изменения словами, не markdown-разметкой
